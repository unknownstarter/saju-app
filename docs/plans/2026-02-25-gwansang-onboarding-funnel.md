## 0. Executive Summary — 왜 관상인가

### 노아님의 핵심 인사이트

> "사진 업로드는 데이팅앱 온보딩의 가장 큰 허들이다.
> 관상을 선물로 주면, 사진 업로드가 의무가 아니라 보상이 된다."

이것은 **프레이밍 전환**(Frame Shift)의 천재적 적용이다:

| 기존 데이팅앱 | 사주인연 (관상 퍼널) |
|-------------|-------------------|
| "프로필 사진을 올려주세요" (의무) | "AI가 당신의 관상을 읽어드릴게요" (선물) |
| 내가 남에게 보여주기 위한 사진 | AI가 나를 읽어주기 위한 사진 |
| 매력어필 압박 → 스트레스 | 운명 발견 재미 → 호기심 |
| 30~40% 완료율 (업계 평균) | **목표: 75%+ 완료율** |

### 시장 적합성 검증

- 한국인 운세 이용 경험 84.5%, 1030세대 90% (기존 PRD 데이터)
- AI 얼평/관상 서비스 이미 한국에서 인기 (ai-face.me, FaceTag 등)
- 사주 + 관상 = "동양 운명학 원스톱" → 시너지 극대화
- MZ세대: 점술을 "반은 믿고, 반은 재미로" 소비하는 세대 → 관상의 엔터테인먼트 가치 높음

### 핵심 메트릭 목표

| 메트릭 | 업계 평균 | 목표 | 비고 |
|--------|----------|------|------|
| 사진 업로드 완료율 | 30~40% | **75%+** | 관상 인센티브 효과 |
| 3장 사진 모두 업로드 | 15~20% | **60%+** | 관상 정확도 프레이밍 |
| 온보딩 전체 완료율 | 40~50% | **70%+** | 사주+관상 이중 보상 |
| 관상 결과 공유율 | N/A (신규) | **25%+** | 바이럴 계수 |
| D1 리텐션 | 25~35% | **50%+** | 결과 확인 욕구 |

## 1. 퍼널 아키텍처 — 관상 통합 설계

### 1.1 현재 퍼널 vs 개선 퍼널

**현재 (Two-Funnel Architecture):**
```
[소셜 로그인] → [Phase A: 사주 퍼널]
                  └─ 이름/성별/생년월일시 → 사주분석 → 사주결과카드
                                                          ↓
              [Phase B: 소개팅 퍼널] ← 매칭 탭 첫 접근 시
                  └─ 사진(2장+) → 기본정보 → 자기표현 → 라이프스타일 → 본인인증
```

**문제점:**
1. Phase B에서 사진 업로드가 "의무"로 느껴짐 (불꼬리: "멋진 사진 보여줘!")
2. 사주 결과 → 매칭 프로필 사이의 감정적 단절 (와우 모먼트 후 갑자기 폼)
3. 사진 업로드에 대한 인센티브 부재

**개선안 (Three-Phase Funnel with 관상):**
```
[소셜 로그인] → [Phase A: 사주 퍼널] (기존과 동일)
                  └─ 이름/성별/생년월일시 → 사주분석 → 사주결과카드
                                                          ↓
              [Phase A+: 관상 퍼널] ← 사주 결과 직후 자연 유입 ★NEW★
                  └─ "사주에 관상까지 더하면 운명이 더 정확해져요"
                  └─ 사진 3장 업로드 (관상용) → AI 관상분석 → 관상결과카드
                  └─ 사주+관상 통합 = "내 운명 카드" → 공유
                                                          ↓
              [Phase B: 소개팅 퍼널] ← 관상 완료 후 자연 전환
                  └─ 사진은 이미 있음! → 기본정보 → 자기표현 → 라이프스타일 → 완료!
```

### 1.2 핵심 설계 원칙

1. **관상은 Phase A와 B 사이의 "브릿지"다** — 사주의 신비로운 감정을 이어가면서, Phase B의 사진을 미리 수집한다
2. **사용자는 "관상을 받기 위해" 사진을 올린다** — "데이팅 프로필용"이라는 인식이 들지 않아야 한다
3. **관상 결과가 사주 결과와 합쳐져야 한다** — 별개의 결과가 아니라 "하나의 운명 카드"
4. **사진 품질이 자연스럽게 보장된다** — "관상 정확도를 위해" 얼굴이 잘 보이는 사진 요구

## 2. 스크린 바이 스크린 온보딩 플로우

### Step 0: 소셜 로그인 (기존)
- Apple / Google / Kakao
- 변경 없음

### Step 1-2: 사주 정보 입력 (기존 Phase A)
- 이름, 성별, 생년월일 (물결이 가이드)
- 생시(시진) 선택 (쇠동이 가이드)
- 변경 없음

### Step 3: 사주 분석 + 결과 (기존)
- 로딩 연출 → 사주 결과 카드
- **추가 변경**: 결과 카드 하단에 관상 유도 CTA 삽입

### ★ Step 4: 관상 브릿지 화면 (NEW)

**트리거**: 사주 결과 카드 확인 후

**화면 구성:**
```
┌─────────────────────────────────────────┐
│                                          │
│  [사주 결과 카드 축소 미리보기]            │  ← 내 사주 카드가 살짝 보임
│                                          │
│  ┌────────────────────────────────┐      │
│  │  🔮 + 📸                       │      │
│  │                                │      │
│  │  사주만으론 아쉽지 않아?        │      │  ← 나무리 말풍선
│  │                                │      │
│  │  "관상(觀相)까지 더하면         │      │
│  │   너의 운명이 더 선명해져"      │      │
│  │                                │      │
│  │  ✦ 첫인상에서 풍기는 기운       │      │  ← 관상으로 알 수 있는 것 3가지
│  │  ✦ 연애할 때 빠지는 매력        │      │
│  │  ✦ 사주+관상 종합 인연 지수     │      │
│  │                                │      │
│  └────────────────────────────────┘      │
│                                          │
│  ┌──────────────────────────────────┐    │
│  │  📸 AI 관상 분석 받기 (무료)     │    │  ← CTA (메인)
│  └──────────────────────────────────┘    │
│                                          │
│         나중에 할게요                     │  ← 스킵 (Phase B로 직행)
│                                          │
└─────────────────────────────────────────┘
```

**사용자 심리 설계:**
| 요소 | 심리 효과 |
|------|----------|
| "사주만으론 아쉽지 않아?" | 불완전함 자극 (Zeigarnik Effect) |
| "무료" 강조 | 손실 회피 — "안 하면 손해" |
| 사주 카드가 옆에 보임 | 이미 투자한 시간 → 매몰비용 효과 |
| 알 수 있는 것 3가지 미리보기 | 구체적 보상 약속 → 호기심 |
| 나무리 캐릭터 안내 | 친숙한 캐릭터의 권유 → 신뢰 |

**예상 전환율**: 사주 완료자의 **70~80%** 가 관상 진행 (나머지는 Phase B로 직행)

### ★ Step 5: 관상용 사진 업로드 (NEW — 핵심 화면)

**왜 3장인가?**

| 사진 | 관상 분석 용도 (표면적 이유) | 데이팅 프로필 용도 (실제 효과) |
|------|---------------------------|---------------------------|
| **1번: 정면 사진** | "이목구비 분석, 오행 기운 파악" | 대표 프로필 사진 (얼굴 확인) |
| **2번: 미소 짓는 사진** | "웃을 때 관상이 진짜 관상! 입꼬리, 눈웃음 분석" | 매력 어필 사진 (밝은 인상) |
| **3번: 전신 또는 반신 사진** | "체형에서 오는 기운, 전체 인상 분석" | 전체 모습 사진 (체형 확인) |

**이 프레이밍이 천재적인 이유:**
- 사용자는 "관상 정확도를 위해" 좋은 사진을 올리려고 노력한다
- "얼굴이 잘 보여야 관상이 정확해요" → 자연스럽게 고품질 얼굴 사진 수집
- "웃는 사진이 진짜 관상" → 매력적인 표정의 사진 확보
- "전신 기운 분석" → 전체 모습 사진까지 자연스럽게 수집

**화면 구성:**
```
┌─────────────────────────────────────────┐
│  ← 뒤로    AI 관상 분석    1/3           │
│                                          │
│  ┌────────────────────────────────┐      │
│  │  불꼬리: "관상은 얼굴에 담긴    │      │
│  │   운명이야! 정면 사진부터~"     │      │
│  └────────────────────────────────┘      │
│                                          │
│  ┌────────────────────────────────┐      │
│  │                                │      │
│  │     ┌─────────────────┐       │      │
│  │     │                 │       │      │
│  │     │   📸 정면 사진   │       │      │  ← 큰 업로드 영역 (200x260)
│  │     │                 │       │      │
│  │     │   [촬영/앨범]    │       │      │
│  │     │                 │       │      │
│  │     └─────────────────┘       │      │
│  │                                │      │
│  │  💡 관상 TIP                   │      │
│  │  "정면을 바라보고 자연스럽게!   │      │
│  │   이목구비가 잘 보여야          │      │
│  │   정확한 관상을 봐줄 수 있어"   │      │
│  │                                │      │
│  └────────────────────────────────┘      │
│                                          │
│  ○ ○ ○  ← 3장 진행 인디케이터           │
│                                          │
│  ┌──────────────────────────────────┐    │
│  │         사진 선택하기             │    │
│  └──────────────────────────────────┘    │
│                                          │
└─────────────────────────────────────────┘
```

**3장 각각의 가이드:**

#### 사진 1: 정면 사진
- **캐릭터**: 불꼬리 (火)
- **말풍선**: "관상은 얼굴에 담긴 운명이야! 정면 사진부터 보여줘~"
- **팁**: "정면을 바라보고 자연스럽게! 이목구비가 잘 보여야 정확한 관상을 봐줄 수 있어"
- **가이드 오버레이**: 얼굴 윤곽 가이드라인 (카메라 실행 시)
- **금지 사항**: 선글라스, 마스크, 과도한 필터 → "관상 정확도가 떨어져요" 경고

#### 사진 2: 미소 짓는 사진
- **캐릭터**: 흙순이 (土)
- **말풍선**: "웃을 때 관상이 진짜 관상이래! 활짝 웃어봐~"
- **팁**: "자연스러운 미소가 최고야! 입꼬리와 눈웃음에서 연애운이 보인대"
- **심리 트리거**: "웃는 얼굴에서 연애운이 보인다" → 최고의 미소 사진을 올리고 싶게 만듦
- **금지 사항**: 무표정 → "살짝이라도 미소 지어봐! 관상이 더 정확해져"

#### 사진 3: 반신/전신 사진 또는 자유 사진
- **캐릭터**: 물결이 (水)
- **말풍선**: "전체적인 기운도 봐야 해! 상반신이 보이는 사진 부탁~"
- **팁**: "상체가 보이는 사진이면 OK! 분위기와 전체 인상에서 오는 기운을 분석해줄게"
- **유연성**: "정면이 아니어도 괜찮아~" → 자연스러운 일상 사진도 허용
- **심리 트리거**: 가장 자신 있는 사진을 자유롭게 올릴 수 있는 기회

**사진 품질 보장 메커니즘:**

| 메커니즘 | 구현 방식 | 사용자 인식 |
|---------|----------|-----------|
| 얼굴 인식 체크 | ML Kit Face Detection | "관상 분석이 가능한 사진인지 확인하고 있어요" |
| 선명도 체크 | 이미지 블러 감지 | "더 선명한 사진이면 관상이 정확해져요" |
| 적절한 크기 | 해상도 최소 기준 | "고화질 사진일수록 세밀한 관상을 봐줄 수 있어요" |
| 필터 경고 | 과도한 보정 감지 | "자연스러운 사진이 관상 정확도를 높여요" |

**핵심: 이 모든 품질 체크가 "관상 정확도"라는 이유로 정당화된다.**
사용자는 데이팅 프로필 사진 품질 요구가 아니라 "관상을 정확하게 받기 위한 조건"으로 인식한다.

### ★ Step 6: AI 관상 분석 로딩 (NEW)

**분위기**: Stage 2(사주 분석)와 동일한 다크 모드, 신비로운 분위기

**로딩 시퀀스 (3~5초, 의도적 지연):**

```
Phase 1 (0~1초): "관상을 읽고 있어요..."
  - 업로드한 3장의 사진이 카드처럼 펼쳐짐
  - 각 사진 위에 오행 기운 아우라가 서서히 나타남

Phase 2 (1~2.5초): "이목구비에서 운명을 찾는 중..."
  - 정면 사진이 중앙으로 오며 확대
  - 눈, 코, 입, 이마, 턱 부위에 분석 포인트 반짝임
  - 각 부위에서 오행 컬러 파티클 방출

Phase 3 (2.5~4초): "사주와 관상을 합치는 중..."
  - 사주 결과 카드 + 관상 분석이 서서히 합쳐지는 모션
  - 두 카드가 빛과 함께 하나로 융합
  - 캐릭터가 "찾았다!" 리액션

Phase 4 (4초~): 결과 카드로 전환
```

**기술 구현**: 실제 AI 분석은 Step 5에서 사진 업로드 시 백그라운드로 시작. 로딩 연출은 순수하게 감정적 경험을 위한 것.

### ★ Step 7: 관상 결과 + 통합 운명 카드 (NEW — 핵심 바이럴 화면)

#### 7.1 관상 결과 구성

**AI가 분석하는 관상 요소:**

| 분석 영역 | 관상학 기반 | AI가 읽는 것 | 결과 문구 예시 |
|----------|-----------|-------------|--------------|
| **이마(天庭)** | 초년운, 지성 | 이마 넓이/높이 비율 | "넓은 이마에서 지적인 기운이 느껴져요" |
| **눈(眼)** | 감정, 의지력 | 눈 크기/간격/모양 | "깊은 눈빛, 한번 빠지면 헤어나올 수 없는 매력" |
| **코(鼻)** | 재물운, 자존심 | 코 길이/너비 | "단단한 코에서 자기 확신의 기운이" |
| **입(口)** | 복록, 표현력 | 입 크기/입꼬리 | "웃을 때 올라가는 입꼬리, 복이 들어오는 상" |
| **턱(顎)** | 말년운, 의지 | 턱 라인/각도 | "각진 턱선에서 강한 의지가 느껴져요" |
| **전체 인상** | 오행 기운 | 종합 분석 | "불의 기운이 강한 열정적인 관상" |

**관상 결과 카드 구조:**
```
┌─────────────────────────────────────────┐
│  [다크 모드 — 신비로운 배경]              │
│                                          │
│  ✨ {이름}님의 관상                       │
│                                          │
```

## 3. 관상 결과의 "납득 가능성" 설계

### 3.1 관상이 "맞다"고 느끼게 만드는 기법

**이것이 사주/관상 서비스의 핵심이다. 랜덤 포춘쿠키처럼 느껴지면 바이럴도 없다.**

#### Barnum Effect (바넘 효과) + 구체적 관찰의 조합

```
❌ 나쁜 예 (너무 일반적):
"당신은 따뜻한 사람입니다"

❌ 나쁜 예 (너무 구체적이라 틀릴 수 있음):
"당신은 3년 전에 큰 실연을 경험했습니다"

✅ 좋은 예 (관찰 + 해석):
"눈꼬리가 살짝 올라가면서도 부드러운 눈매를 가지셨네요.
이런 관상은 겉으론 도도해 보이지만,
막상 알고 보면 따뜻한 사람이라는 말을 자주 듣는 타입이에요."
```

**패턴**: `실제 얼굴 특징 관찰` + `그로부터 유추되는 성격` + `일상적 경험과 연결`

이 구조가 "납득 가능"한 이유:
1. **실제 관찰**: "눈꼬리가 올라간다" → 사진에서 확인 가능
2. **전통 해석**: "이런 관상은 ~한 성격" → 관상학 권위
3. **일상 연결**: "~라는 말을 자주 듣는 타입" → 거의 모든 사람에게 해당되지만 "맞아!" 느낌

#### AI 관상 결과 생성 구조

```
Input:
  - 얼굴 특징 데이터 (ML Kit / Vision API):
    - 눈 크기, 간격, 각도
    - 코 길이, 너비
    - 입 크기, 입꼬리 각도
    - 이마 넓이/높이
    - 턱 라인 각도
    - 얼굴형 (둥근/각진/계란형 등)
  - 사주 데이터:
    - 일주, 주도 오행, 오행 밸런스

Processing (Claude API):
  프롬프트: "다음 얼굴 특징과 사주 데이터를 기반으로,
  한국 전통 관상학에 입각한 관상 해석을 생성하세요.
  규칙:
  1. 반드시 실제 얼굴 특징을 언급할 것 (관찰 가능한 사실)
  2. 관상학적 해석을 덧붙일 것 (전통 권위)
  3. 사주 오행과의 시너지를 연결할 것
  4. 부정적 해석은 긍정적으로 리프레이밍할 것
  5. 연애/인간관계 관련 해석을 강조할 것 (데이팅 앱 맥락)
  6. 톤: 부드럽고 신비로우면서도 구체적"

Output:
  - 부위별 관상 (눈/코/입/이마/턱) × (점수 + 한 줄 해석)
  - 종합 관상 스토리 (3~5문장)
  - 관상 기반 연애 성향 (2~3줄)
  - 관상 궁합 잘 맞는 유형 (1~2줄)
  - 매력 키워드 3개
```

### 3.2 결과 구조 — 무료 vs 프리미엄

| 항목 | 무료 (기본 관상) | 프리미엄 (상세 관상) |
|------|----------------|-------------------|

## 5. 관상 궁합 — 매칭에서의 활용

### 5.1 관상 궁합 점수

사주 궁합에 더해 관상 궁합을 추가하면:

```
총 궁합 = (사주 궁합 × 0.6) + (관상 궁합 × 0.2) + (취향 매칭 × 0.2)
```

**관상 궁합 요소:**
| 요소 | 가중치 | 근거 |
|------|--------|------|
| 오행 조화 (얼굴 기운 + 사주 오행) | 40% | 전통 관상학: 상생 얼굴 조합 |
| 인상 보완성 (부드러운 + 강한) | 30% | 관상학: 음양 조화 원리 |

### 6.1 사용자 반발 위험

| 위험 | 심각도 | 확률 | 대응 |

### 6.2 외모 점수화 이슈 — 극도로 주의

**절대 하면 안 되는 것:**
- "외모 점수 78점" 같은 직접적 외모 평가
- "당신은 상위 23%의 외모" 같은 순위화
- 부정적 관상 표현 ("이런 코는 재물운이 나쁩니다")

**반드시 해야 하는 것:**
- "매력도", "호감도", "연애운" 등 운명/기운 프레이밍
- 모든 결과를 긍정적으로 (부정적 요소도 "극복 가능한 과제"로)
- "관상은 변하는 것" 메시지 포함
- "AI 분석은 재미로 참고하세요" 면책 문구

### 6.3 기술 위험

| 위험 | 대응 |
|------|------|

## 10. 경쟁 해자 분석

### 10.1 왜 다른 앱이 따라하기 어려운가

| 해자 | 설명 |
|------|------|
| **사주+관상 이중 분석** | 서양 앱은 관상을 Physiognomy로 접근 (타부시), 한국은 문화적 수용도 높음 |
| **오행 연결 시스템** | 사주의 오행과 관상의 인상을 하나의 프레임으로 연결하는 것은 사주인연만의 고유 자산 |
| **캐릭터 + 관상** | 오행이 캐릭터가 관상을 안내하는 UX는 단순 관상 앱과 차별화 |
| **데이팅 퍼널 통합** | 관상 → 데이팅 전환의 자연스러운 브릿지는 순수 점술 앱이나 순수 데이팅 앱에는 없음 |

### 10.2 기존 AI 관상 앱과의 차별점

| 기존 AI 관상 (ai-face.me, FaceTag) | 사주인연 관상 |
|-----------------------------------|-------------|
| 단독 서비스 (관상만) | 사주+관상+매칭 통합 경험 |
| 외모 점수 중심 | 운명/기운 프레이밍 (외모 점수 없음) |
| 1회성 사용 | 매칭 루프에 지속적으로 활용 |
| 바이럴만 있고 리텐션 없음 | 관상 → 데이팅 → 리텐션 루프 |
| 수익 모델 미약 (광고) | 프리미엄 관상 + 관상 궁합 결제 |

## 11. 마무리 — 왜 이 퍼널이 작동하는가

### Pirate Metrics (AARRR) 관점

| 단계 | 관상이 기여하는 것 |
|------|------------------|
| **Acquisition** | "AI가 관상 봐준다" → 호기심 유입 + 바이럴 카드 공유 |
| **Activation** | 사진 3장 업로드 = 프로필 완성의 가장 큰 허들 해결 |
| **Retention** | 관상 결과 업데이트 ("사진 바꾸면 관상이 달라질 수도?") |
| **Revenue** | 프리미엄 관상 리포트 (300P), 관상 궁합 상세 (500P) |
| **Referral** | 운명 카드 SNS 공유, "너도 관상 봐봐" 바이럴 루프 |

### 한 문장 요약

> **관상은 사용자에게는 "선물"이고, 서비스에게는 "프로필 사진 수집 엔진"이며, 시장에는 "바이럴 카드"다.**
