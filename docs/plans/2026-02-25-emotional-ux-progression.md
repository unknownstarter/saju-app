# Emotional UX Progression — 사주인연

> "운명적 만남"이라는 내러티브를 6단계 감정 곡선으로 설계.
> 각 단계는 이전 단계의 감정을 먹고 자란다. 스킵 불가.

---

## Emotion Curve Overview

```
감정 강도
  ▲
  │                                          ★ Match Success
  │                                         ╱ ╲
  │                                        ╱   ╲
  │                    ◆ First Match       ╱     ╲  💎 Purchase
  │                   ╱  Preview          ╱       ╲╱
  │        ◈ Saju    ╱                   ╱
  │       Calc      ╱        ♡ Like     ╱
  │      ╱    ╲    ╱         Sending   ╱
  │     ╱      ╲  ╱             │     ╱
  │ ○  ╱        ╲╱              │    ╱
  │ First                       │   ╱
  │ Open                        │  ╱
  │                             │ ╱
  └──────────────────────────────────────── 시간
   안정    긴장  신비   호기심  두근  폭발  성찰
```

핵심 원칙:
- **이완 → 긴장 → 해소** 사이클을 2번 반복 (사주 계산 → 매칭 프리뷰 / 좋아요 → 매치 성공)
- 두 번째 사이클이 더 강하다 (기대감 누적)
- 결제는 감정 피크 직후, "더 알고 싶다"는 욕구가 최대일 때

## Stage 1: First Open — "여기 뭔가 다르다"

### Emotion Target
**고요한 호기심** — 여느 데이팅앱과 다른 분위기. 신비롭지만 무겁지 않고, 세련되었지만 차갑지 않다.
"이건 뭐지?" → "오, 예쁘다" → "좀 더 보고 싶다"

### Visual Tone
| 요소 | 설정 |
|------|------|
| 배경 | 한지 베이지(`hanjiBg`) — 따뜻하고 익숙한 질감 |
| 타이포 | Pretendard 36px, w700 — "사주로 만나는 운명" 한 줄 |
| 색감 | 모노톤 + 오행 컬러 한 점 (waterColor 원형이 스르륵 등장) |
| 캐릭터 | 나무리 64px, 살짝 기울인 고개 — 호기심 유발 |
| 레이아웃 | 중앙 정렬, 넉넉한 여백(64px+), 요소 3개 이하 |

### Animation Tempo
**느리고 부드러움** — 1000~1500ms, easeOutCubic
- 로고 fadeIn: 800ms (opacity 0→1)
- 태그라인 slideUp: 1000ms (y: 20→0, stagger 200ms per word)
- 캐릭터 등장: 1200ms (scale 0.8→1.0, elasticOut)
- 전체 분위기: **숨을 쉬는 듯한 리듬**

### Sound Feedback
**없음 (의도적 침묵)**
- 첫 화면은 소리 없이 시각만으로 분위기를 전달
- 사용자가 "소리가 날 것 같은" 기대감을 갖게 만든다
- 유일한 피드백: CTA 버튼 탭 시 `HapticFeedback.lightImpact()`

### Monetization Pressure
**0/5 — 제로**
- 결제 언급, 프리미엄 뱃지, 가격 표시 일절 없음
- 오직 "이 세계에 들어오세요"라는 초대만
- 목표: 이탈률 < 30% (첫 화면에서 떠나지 않게)

## Stage 2: Saju Calculation — "내 운명이 펼쳐진다"

### Emotion Target
**경이로운 긴장감** — 생년월일시를 입력하는 순간, 진지한 의식(儀式)이 시작된다.
"내 사주가 뭘까?" → (입력) → "와, 이게 나야?" → "신기하다..."

### Visual Tone
| 요소 | 설정 |
|------|------|
| 배경 | **다크 모드 자동 전환** — inkBlack + 미세한 별 파티클 |
| 타이포 | 사주 결과: 48px display, w700, mysticGlow 컬러 |
| 색감 | 사용자 오행에 맞는 컬러가 서서히 등장 (木→초록 그라데이션) |
| 캐릭터 | 해당 오행 캐릭터가 "찾았다!" 말풍선과 함께 등장 |
| 레이아웃 | 세로 스크롤, 카드 단위 정보 공개 (한 번에 하나씩) |

### Animation Tempo
**중간 → 극적 가속** — 계산 중은 느리게, 결과 공개는 빠르게
- 입력 후 로딩: 2000~3000ms (의도적 지연 — "계산하고 있어요")
  - 팔자 글자가 하나씩 떠오르는 모션 (250ms stagger)
  - 오행 바퀴가 천천히 회전
- 결과 공개: 500ms snap (tension release)
  - 점수/프로필이 한꺼번에 "딱" 나타남
  - 캐릭터 bounce (elasticOut, 400ms)
- **핵심: 기다림이 기대감을 만든다**

### Sound Feedback
**미세한 피드백 시작**
- 입력 완료 시: `HapticFeedback.mediumImpact()` — "접수됐다"
- 계산 중: 미세한 진동 패턴 (100ms on, 200ms off, 3회) — "뭔가 일어나고 있다"
- 결과 공개 시: `HapticFeedback.heavyImpact()` — "짠!"
- 사운드 이펙트: 종소리 한 번 (선택적, 토글 가능) — 0.3초, 여운 있는 맑은 소리

### Monetization Pressure
**1/5 — 씨앗 심기**
- "기본 사주 분석"은 완전 무료
- 결과 하단에 흐릿한 잠금 섹션: "심화 분석 (오행 밸런스, 대운 흐름)"
- 가격 표시 없음 — 단지 "더 있다"는 것만 암시
- 목표: "이것도 궁금한데..." 라는 마음의 씨앗

## Stage 3: First Match Preview — "이 사람... 혹시?"

### Emotion Target
**두근거리는 호기심** — 처음으로 "상대방"이라는 존재를 만난다.
"오, 누가 있지?" → "이 사람 궁합이 87%?" → "더 보고 싶다"

### Visual Tone
| 요소 | 설정 |
|------|------|
| 배경 | 다크 → 살짝 밝아짐 (inkSurface) — 신비 + 설렘 공존 |
| 타이포 | 이름: 18px w600, 궁합 점수: 48px w700 (compatibilityExcellent) |
| 색감 | 상대 오행 컬러 + 내 오행 컬러가 만나는 그라데이션 |
| 카드 | MatchCard 등장 — 사진 60% + 정보 40%, 프리미엄은 금색 테두리 |
| 레이아웃 | 카드 1장씩 중앙 배치 (스와이프 아님!), 충분한 여백 |

### Animation Tempo
**기대감 있는 중간 속도** — 500~800ms
- 카드 등장: slideUp(24px) + fadeIn, 600ms, easeOutQuint
- 궁합 점수: countUp 0→87, 1200ms (easeOutCubic) — Stage 2의 극적 공개 반복
- 강점 칩: stagger fadeIn, 200ms per chip, 100ms delay
- **카드 간 전환**: 300ms crossFade (빠르지만 급하지 않게)

### Sound Feedback
**심장박동 리듬**
- 카드 등장 시: `HapticFeedback.mediumImpact()` — "두근"
- 점수 카운트업 중: 미세한 진동이 점점 강해짐 (점수에 비례)
- 90점 이상일 때: `HapticFeedback.heavyImpact()` + 0.1초 후 한 번 더 — "쿵쿵!"
- 사운드: 부드러운 차임 (점수 공개 완료 시, 0.5초)

### Monetization Pressure
**2/5 — 호기심 자극**
- 궁합 점수는 보여주되, "왜 잘 맞는지"는 블러 처리
- CompatibilityCard에 "상세 궁합 보기 →" CTA (가격 미표시)
- 프리미엄 매치는 금색 테두리로 자연스럽게 차별화
- 하루 무료 추천 N명 표시: "오늘의 인연 3/5"
- 목표: "점수만으론 부족해, 자세히 알고 싶어"

## Stage 4: Sending Like — "이 마음을 보낼까..."

### Emotion Target
**떨리는 결단** — 행동을 취하는 순간의 긴장. "보낼까 말까" 고민 후 버튼을 누르는 용기.
"좋아요를 보내면... 알게 되나?" → (탭) → "보냈다!" → "어떻게 될까..."

### Visual Tone
| 요소 | 설정 |
|------|------|
| 배경 | 카드 디테일 뷰 — 상대 프로필이 크게 보임 |
| LikeButton | 로즈핑크(`compatibilityExcellent`), 52px 높이, 중앙 하단 고정 |
| PremiumLikeButton | 금색 그라데이션, "300P" 뱃지 항상 노출, LikeButton 위에 배치 |
| 전송 중 | 버튼 내 스피너, 텍스트 "전송 중..." — 0.5~1초 의도적 딜레이 |
| 전송 완료 | 하트 아이콘 스케일 바운스 + "보냈어요 ♥" 텍스트 스왑 |

### Animation Tempo
**짧고 강한 리듬** — 100~400ms (결단의 순간은 빨라야 함)
- 버튼 press: 100ms scale(0.95) — 즉각 반응
- 전송 중: 500ms spinner (너무 빠르면 가볍게 느낌)
- 성공 모션: 400ms (heart scale 1.0→1.3→1.0, elasticOut)
- 텍스트 스왑: 200ms crossFade
- **전체 시퀀스: press(100ms) → load(500ms) → success(400ms) = ~1초**

### Sound Feedback
**결단 → 보상**
- 버튼 press: `HapticFeedback.mediumImpact()` — "찰칵" (결단 확인)
- 전송 완료: `HapticFeedback.heavyImpact()` — "쿵!" (보상감)
- 포인트 부족 시: `HapticFeedback.heavyImpact()` (경고성) — "안 돼!"
- 사운드: 하트 날아가는 "뿅" (0.3초, 밝고 가벼운 톤)

### Monetization Pressure
**3/5 — 자연스러운 선택지**
- 무료 좋아요: "오늘 2/3 남음" 서브텍스트 (소진감)
- 슈퍼좋아요: "상대에게 바로 전달돼요" (가치 명시) + "300P" (비용 투명)
- 무료 소진 후: LikeButton이 "좋아요 추가 구매 (100P)"로 변형
- 핵심 심리: **"이 사람한테는 슈퍼좋아요를 보내야 하지 않을까?"**
- 목표: 슈퍼좋아요 전환율 > 15% (일반 좋아요 대비)

## Stage 5: Match Success — "운명이었어!"

### Emotion Target
**폭발적 환희** — 앱 전체에서 가장 강한 감정적 순간. 도파민 최대.
"♡ 매칭됐어요!" → "세상에!" → "어떤 사람이지?!" → "대화하고 싶다!"

### Visual Tone
| 요소 | 설정 |
|------|------|
| 배경 | **풀스크린 오버레이** — 반투명 다크 + 중앙에 매칭 카드 |
| 색감 | 두 사람의 오행 컬러가 섞이는 그라데이션 (木+火 = 초록→분홍) |
| 타이포 | "천생연분!" 32px, w700 — 궁합 등급에 따른 문구 |
| 파티클 | 오행 컬러 별/꽃잎 파티클 (8~12개, 2초간 서서히 떨어짐) |
| 양쪽 프로필 | 두 원형 아바타가 중앙에서 만남 (좌→우, 우→좌) |
| CTA | "대화 시작하기" 버튼 — 궁합 점수 배경색 |

### Animation Tempo
**폭발 → 여운** — 시퀀스가 핵심
1. **빌드업** (0~300ms): 화면 약간 어두워짐, 0.2초 정적
2. **폭발** (300~800ms): 두 아바타가 중앙으로 슬라이드 (300ms, easeOutBack) → 만나는 순간 파티클 burst + 스케일 바운스
3. **텍스트 등장** (800~1200ms): "천생연분!" fadeIn + slideUp, 400ms
4. **여운** (1200~2500ms): 파티클이 천천히 떨어짐, 궁합 점수 countUp
5. **CTA 등장** (2500ms~): "대화 시작하기" fadeIn

**총 시퀀스: ~3초 (스킵 불가, 하지만 길지 않게)**

### Sound Feedback
**축하 시퀀스**
- 아바타 충돌 순간: `HapticFeedback.heavyImpact()` — "쿵!"
- 파티클 burst: 연속 진동 (50ms × 3, 30ms 간격) — "두두둑"
- 텍스트 등장: `HapticFeedback.mediumImpact()` — "짠"
- 사운드: 맑은 차임 + 미세한 반짝임 효과 (1.5초, 앱 내 가장 화려한 소리)
- **이 순간의 소리가 앱의 시그니처 사운드가 된다**

### Monetization Pressure
**2/5 — 의도적 완화**
- 매칭 성공 순간에는 결제 유도 최소화
- 감정적 피크를 훼손하지 않음 → 순수한 기쁨
- 단, 하단에 아주 작게: "궁합 상세 리포트도 준비되어 있어요" (텍스트만, CTA 아님)
- **핵심: 이 기쁨을 기억하게 만들어야 다음 매칭을 위해 돌아온다**
- 목표: 매칭 후 채팅 진입율 > 80%, D1 리텐션 > 60%

## Stage 6: Detailed Compatibility Purchase — "우리 관계를 더 알고 싶어"

### Emotion Target
**깊은 궁금증 + 합리화** — 감정적 피크(Stage 5) 이후 "왜?"라는 질문이 온다.
"왜 우리가 잘 맞는 거지?" → "상세 분석이 있네" → "500P면... 커피 한 잔인데" → (구매)

### Visual Tone
| 요소 | 설정 |
|------|------|
| 배경 | 다크 모드 유지 — 신비로운 톤, "비밀을 풀어보세요" |
| 프리뷰 | 상세 리포트의 첫 20%만 공개 (나머지 블러) |
| 블러 섹션 | 가우시안 블러 8px + "잠금" 아이콘 + mysticGlow 테두리 |
| CTA | "궁합 상세 리포트 열기 — 500P" 골드 그라데이션 버튼 |
| 신뢰 요소 | "10,847명이 확인했어요" 소셜 프루프 |
| 리포트 | 오행 상생/상극 차트, 일주 궁합, AI 스토리텔링 해석 |

### Animation Tempo
**차분하고 설득적** — 700~1000ms
- 프리뷰 섹션: 스크롤 기반 reveal (parallax, 부드럽게)
- 블러 섹션: 살짝 흔들리는 듯한 모션 (breathe, 2초 주기) — "열 수 있을 것 같은데..."
- 구매 후 언락: 블러가 위에서 아래로 걷히는 모션 (800ms, easeOutQuint)
- 리포트 카드: stagger fadeIn (200ms per section)
- **템포 핵심: 급하게 느끼지 않게. "천천히, 깊게"**

### Sound Feedback
**고급스러운 피드백**
- 스크롤 중 블러 섹션 도달: 미세한 `HapticFeedback.lightImpact()` — "여기 잠겨있어요"
- 구매 버튼 탭: `HapticFeedback.mediumImpact()`
- 구매 완료 + 언락: `HapticFeedback.heavyImpact()` — "열렸다!"
- 사운드: 봉인이 풀리는 듯한 소리 (0.8초, 낮은 톤 → 맑은 톤으로 상승)

### Monetization Pressure
**4/5 — 최대 (하지만 우아하게)**
- 감정적 정당화 완료된 시점 (Stage 5의 기쁨 → "더 알고 싶다")
- 프리뷰로 가치 증명 후 구매 유도
- 가격 앵커링: "커피 한 잔 가격으로 운명을 읽어보세요"
- 소셜 프루프: "이 궁합을 확인한 커플 87%가 대화를 이어갔어요"
- 시간 한정: "매칭 후 24시간 내 50% 할인" (긴급성)
- 무료 미끼: 첫 궁합 리포트 1회 무료 (맛보기 → 습관화)
- **절대 하지 않을 것: 팝업, 강제 광고, 기능 차단** → 신뢰 훼손

## Cross-Stage Design Rules

### 1. 다크 모드 전환 타이밍
```
Stage 1 (First Open)     → Light  — 편안한 진입
Stage 2 (Saju Calc)      → Dark   — 신비로운 의식
Stage 3 (Match Preview)  → Dark   — 운명적 분위기 유지
Stage 4 (Sending Like)   → Dark   — 긴장감 유지
Stage 5 (Match Success)  → Dark   — 감정적 피크
Stage 6 (Purchase)       → Dark   — 프리미엄 느낌

일상 화면 (홈 피드, 설정, 프로필 편집) → Light
```

### 2. 햅틱 강도 에스컬레이션
```
light → medium → heavy → heavy×2 → burst(3) → heavy
 S1       S2       S3       S3(90+)     S5        S6
```
감정 강도에 비례. Stage 5가 최대.

### 3. 애니메이션 속도 패턴
```
느림(1000ms) → 중간(500ms) → 빠름(100ms) → 느림(3000ms시퀀스) → 차분(800ms)
   S1             S2/S3          S4              S5                  S6
```
Stage 4(결단)가 가장 빠르고, Stage 5(폭발)는 시퀀스가 길지만 각 요소는 빠름.

### 4. 색감 온도 흐름
```
따뜻한 베이지 → 차가운 다크+별 → 오행 컬러 등장 → 핑크/골드 강조 → 두 색 혼합 → 골드 프리미엄
     S1              S2              S3               S4              S5           S6
```

### 5. 정보 공개 전략
```
0% 공개(분위기만) → 내 사주(자기인식) → 상대 프리뷰(타자인식) → 행동(결단) → 결과(보상) → 심화(유료)
       S1                S2                  S3                  S4          S5           S6
```
각 단계마다 "조금 더" 공개 → 다음 단계로의 자연스러운 욕구.

### 6. 모네타이제이션 압력 곡선
```
0  →  1  →  2  →  3  →  2  →  4
S1    S2    S3    S4    S5    S6
```
**Stage 5(매칭 성공)에서 의도적으로 낮춘다.** 감정적 피크를 돈으로 오염시키면 앱 전체 신뢰가 무너진다. 대신 Stage 6에서 자연스럽게 전환.
