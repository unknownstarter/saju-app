# Product Designer — 사주 데이팅 앱

## 1. 디자인 원칙

### 핵심 원칙 5가지

#### 1) Trust First (신뢰 우선)
- 데이팅 앱에서 신뢰는 생명. 모든 디자인 결정에서 "이것이 사용자에게 안전하다고 느껴지는가?"를 질문
- 실제 사진 인증 배지, 프로필 완성도 표시, 신고 버튼 접근성
- 사주 분석의 신뢰도: 출처 표기, "전통 명리학 기반" 문구, 과장된 표현 금지

#### 2) Delight Through Discovery (발견의 즐거움)
- 사주 결과를 보는 순간의 "오!" 모먼트가 앱의 핵심 경험
- 궁합 점수를 확인하는 설렘
- 매일 달라지는 오늘의 운세
- 예상치 못한 매칭의 기쁨

#### 3) Simplicity Over Complexity (단순함 우선)
- 사주는 복잡한 학문이지만, UI는 단순해야 함
- 전문 용어를 일상 언어로 번역
- 한 화면에 하나의 핵심 행동만 요구
- 정보의 점진적 공개 (Progressive Disclosure)

#### 4) Emotional Design (감성 디자인)
- 색상, 일러스트레이션, 애니메이션으로 감정 전달
- 오행의 색상 체계를 디자인 시스템에 녹여냄
- 매칭 성사 시 축하 애니메이션
- 사주 결과의 긍정적 프레이밍

#### 5) Safety by Design (안전 설계)
- 차단/신고는 항상 2탭 이내 접근 가능
- 민감한 정보(생년월일시)는 상대에게 직접 노출하지 않고 궁합 결과로만 표시
- 스크린샷 감지 알림 (채팅 화면)
- 프로필 사진 다운로드 방지

## 2. 사주 시각화 패턴

### 오행(五行) 시각화

#### 컬러 시스템
```
목(木) = #4CAF50 (Green 500) — 성장, 생명력
화(火) = #F44336 (Red 500) — 열정, 에너지
토(土) = #FFC107 (Amber 500) — 안정, 중심
금(金) = #9E9E9E (Grey 500) — 결단, 정제
수(水) = #2196F3 (Blue 500) — 지혜, 유연
```

#### 오행 비율 차트
```
레이더 차트 (Pentagon):
  - 5개 축: 목, 화, 토, 금, 수
  - 사용자의 사주에서 각 오행 비율을 시각화
  - 부드러운 그라데이션 fill
  - 탭하면 각 오행 설명 표시

도넛 차트 (대안):
  - 5개 섹션, 각 오행 색상
  - 중앙에 일간(日干) 한자 표시
  - 간결하고 직관적

바 차트 (가장 단순):
  - 수평 바 5개
  - 각 오행 비율 퍼센트 표시
  - 프로필 카드에 적합한 compact 형태
```

#### 궁합 시각화
```
이중 레이더 차트:
  - 내 오행 + 상대 오행을 같은 차트에 오버레이
  - 겹치는 부분과 보완 부분 시각적 구분
  - 탭하면 "목+화 = 상생 관계" 같은 설명

궁합 점수 게이지:
  - 원형 게이지 (0-100%)
  - 색상: 50 이하 = 오렌지, 50-80 = 하늘색, 80+ = 핑크/하트
  - 점수 아래 한줄 요약 (예: "서로의 부족한 부분을 채워주는 관계")

상생/상극 다이어그램:
  - 오행 순환 다이어그램에서 두 사람의 관계 화살표 표시
  - 상생(→) = 초록, 상극(→) = 빨강
  - 쉬운 언어로 관계 설명
```

#### 사주팔자 원국 표시
```
┌──────────┬──────────┬──────────┬──────────┐
│  시주     │  일주     │  월주     │  연주     │
│ (時柱)   │ (日柱)   │ (月柱)   │ (年柱)   │
├──────────┼──────────┼──────────┼──────────┤
│  천간     │  천간     │  천간     │  천간     │
│  (天干)   │ ★일간★  │  (天干)   │  (天干)   │
├──────────┼──────────┼──────────┼──────────┤
│  지지     │  지지     │  지지     │  지지     │
│  (地支)   │  (地支)   │  (地支)   │  (地支)   │
└──────────┴──────────┴──────────┴──────────┘

디자인 가이드:
- 각 칸에 한자 + 한글 병기
- 오행 색상으로 배경 컬러링
- 일간(日干)을 가장 강조 (굵은 테두리 또는 하이라이트)
- 초보자용: 한글 위주 + 설명 tooltip
- 매니아용: 한자 위주 + 십신 표시 토글
```

## 3. 온보딩 플로우 설계

### 원칙
- **3분 이내 첫 가치 전달** (사주 결과 확인)
- **5분 이내 프로필 완성**
- **불필요한 단계 최소화** — 매칭에 필수인 정보만 온보딩에서 수집
- **점진적 프로필 완성** — 나머지는 사용하면서 채우도록 넛지

### 온보딩 스텝

```
Step 1: 스플래시 & 가치 제안 (5초)
  - "사주로 찾는 나의 인연"
  - 아름다운 오행 애니메이션 배경
  - "시작하기" CTA

Step 2: 소셜 로그인 (10초)
  - 카카오 로그인 (메인, 한국 시장 1순위)
  - Apple 로그인 (iOS 필수)
  - 전화번호 로그인 (대안)
  - 구글 로그인
  - 약관 동의 바텀시트 (전체 동의 + 개별 토글)

Step 3: 생년월일시 입력 ★핵심★ (30초)
  - 날짜: 네이티브 Date Picker (양력 기본, 음력 전환 가능)
  - 시간: 시간대 선택 (子時~亥時, 12시진 + 현대 시간 병기)
  - "태어난 시간을 모르겠어요" 옵션 (시주 제외 분석)
  - 성별 선택 (사주 분석에 필요)

Step 4: 사주 분석 결과 ★WOW 모먼트★ (1분)
  - 로딩: 오행 파티클 애니메이션 (3초)
  - 결과 카드:
    - 일간 기반 성격 유형 (예: "丙火 — 태양처럼 밝은 당신")
    - 오행 비율 차트
    - 핵심 성격 키워드 3개
    - 연애 스타일 한줄 요약
  - "프로필 만들기" CTA + "SNS에 공유하기" 서브 CTA

Step 5: 기본 프로필 (1분)
  - 닉네임 입력
  - 사진 1장 이상 업로드 (갤러리/카메라)
  - 키, 직업 카테고리, 위치 (동네 수준)
  - 자기소개 (선택, 사주 기반 자동 생성 제안)

Step 6: 매칭 선호도 (30초)
  - 관심 성별
  - 선호 나이대 (슬라이더)
  - 선호 거리 (슬라이더)
  - "사주 궁합 중시" 토글

Step 7: 완료 & 첫 추천 (즉시)
  - 축하 애니메이션
  - "첫 번째 인연을 찾아볼까요?"
  - 바로 매칭 화면으로 이동
```

### 온보딩 최적화 규칙
- 각 스텝에 프로그레스 바 표시
- "건너뛰기" 허용하되, 필수 항목(생년월일, 성별, 사진)은 제외
- 이탈 시점 추적: 어느 스텝에서 이탈하는지 Analytics 필수
- A/B 테스트: Step 3(사주 입력)과 Step 5(프로필)의 순서 테스트

## 4. 프로필 카드 디자인

### 카드 구조
```
┌─────────────────────────┐
│                         │
│    [프로필 사진]          │
│    (풀 블리드, 4:5)      │
│                         │
│                         │
├─────────────────────────┤
│ 닉네임, 28              │
│ 丙火 · 서울 강남         │
│                         │
│ ┌─────────────────────┐ │
│ │ 궁합 87%  ❤️‍🔥       │ │
│ │ "서로를 빛나게 하는"   │ │
│ └─────────────────────┘ │
│                         │
│ #열정적 #리더십 #창의적   │
│                         │
│ "여행과 맛집 탐방을      │
│  좋아합니다"             │
│                         │
│   [💚 좋아요]  [❌ 패스]  │
└─────────────────────────┘
```

### 카드 상세 (스크롤/탭 시)
```
기본 정보:
  - 키, 직업, 위치
  - 음주/흡연/종교/MBTI

사주 섹션:
  - 오행 미니 차트
  - 성격 키워드 (사주 기반)
  - 연애 스타일

추가 사진:
  - 최대 6장, 수평 스크롤
  - 사진마다 간단한 캡션 가능

자기소개:
  - 최대 500자
  - 질문-답변 형식 옵션 (예: "주말에 주로 뭐 해요?")
```

### 카드 인터랙션
- **우측 스와이프 또는 하트 버튼**: 좋아요
- **좌측 스와이프 또는 X 버튼**: 패스
- **위로 스와이프**: 슈퍼좋아요 (프리미엄)
- **카드 탭**: 상세 프로필 보기
- **궁합 점수 탭**: 상세 궁합 분석 (프리미엄)

- 그림자: 라이트 모드에서만 사용, 다크 모드에서는 border로 대체

## 6. 접근성 (Accessibility)

### 필수 기준
- **대비율**: 텍스트 4.5:1 이상 (WCAG AA)
- **터치 타겟**: 최소 44x44pt
- **Semantic Labels**: 모든 인터랙티브 요소에 의미 있는 라벨
- **스크린 리더**: VoiceOver(iOS), TalkBack(Android) 지원
- **폰트 크기**: 시스템 Dynamic Type 지원
- **모션**: Reduce Motion 설정 시 애니메이션 비활성화

### 사주 시각화 접근성
```
- 차트에 대한 텍스트 대안 필수
  예: "오행 분포: 목 30%, 화 25%, 토 15%, 금 20%, 수 10%"
- 색상만으로 정보를 전달하지 않음 (패턴 or 라벨 병행)
- 궁합 점수에 숫자 + 텍스트 설명 병행
```

## 7. 디자인 시스템

### Typography

#### 한국어 폰트 스택
```
Primary: Pretendard (본문, UI)
  - 이유: 가변 폰트, 깔끔한 고딕, 숫자/영문 조화
  - Weights: 400(Regular), 500(Medium), 600(SemiBold), 700(Bold)

Secondary: Noto Serif KR (사주 해석, 특별한 텍스트)
  - 이유: 격식 있는 느낌, 사주/운세 콘텐츠에 적합
  - Weights: 400, 700

Accent: 국립박물관문화재단 클래식 (한자, 강조)
  - 이유: 전통적 느낌, 사주팔자 한자 표시에 적합
  - 대안: Nanum Myeongjo
```

#### 타이포 스케일
```
displayLarge:  32px / Bold / -0.5 tracking
displayMedium: 28px / Bold / 0 tracking
headlineLarge: 24px / SemiBold / 0 tracking
headlineMedium: 20px / SemiBold / 0.15 tracking
titleLarge:    18px / Medium / 0.15 tracking
titleMedium:   16px / Medium / 0.15 tracking
bodyLarge:     16px / Regular / 0.5 tracking
bodyMedium:    14px / Regular / 0.25 tracking
bodySmall:     12px / Regular / 0.4 tracking
labelLarge:    14px / Medium / 0.1 tracking
labelMedium:   12px / Medium / 0.5 tracking
labelSmall:    10px / Medium / 0.5 tracking
```

#### 한국어 타이포 규칙
- 줄간격 (line height): 본문 1.6배, 제목 1.3배
- 자간 (letter spacing): 한글은 0 또는 약간 양수(0.15~0.5px)
- 단어 단위 줄바꿈 (word-break: keep-all)
- 숫자/영문 혼용 시 폰트 일관성 유지 (Pretendard가 영문도 지원)
- 최소 폰트 사이즈: 11px (접근성)

### Spacing Scale
```
4px  (xs)   — 아이콘과 텍스트 사이
8px  (sm)   — 관련 요소 간 간격
12px (md)   — 섹션 내 요소 간격
16px (lg)   — 카드 내 패딩
20px (xl)   — 섹션 간 간격
24px (2xl)  — 주요 섹션 간격
32px (3xl)  — 화면 상하 패딩
48px (4xl)  — 대형 간격
```

### Border Radius
```
4px  — 작은 요소 (태그, 뱃지)
8px  — 입력 필드, 버튼
12px — 카드, 바텀시트
16px — 모달, 큰 카드
9999px — 원형 (아바타, 원형 버튼)
```

### Component Library

#### 버튼
```
Primary Button:
  - 배경: brandPrimary
  - 텍스트: white
  - 높이: 52px
  - 테두리 반경: 12px
  - 폰트: titleMedium / SemiBold
  - 사용: CTA, 주요 행동

Secondary Button:
  - 배경: transparent
  - 테두리: brandPrimary 1.5px
  - 텍스트: brandPrimary
  - 높이: 52px
  - 사용: 보조 행동

Ghost Button:
  - 배경: transparent
  - 텍스트: textSecondary
  - 높이: 44px
  - 사용: 건너뛰기, 취소

Icon Button:
  - 크기: 44x44px (터치 타겟)
  - 아이콘 크기: 24px
  - 사용: 좋아요, 차단, 설정 등
```

#### 입력 필드
```
Text Field:
  - 높이: 52px
  - 배경: bgTertiary
  - 테두리: 포커스 시 brandPrimary 2px
  - 패딩: 좌우 16px
  - 라벨: 플로팅 라벨 패턴
  - 에러: 하단에 빨간 텍스트 + 아이콘
```

#### 카드
```
Profile Card:
  - 사진: 4:5 비율, 상단 풀블리드
  - 정보: 하단 패딩 16px
  - 그림자: 0 2px 8px rgba(0,0,0,0.08)
  - 테두리 반경: 16px
  - 스와이프 제스처 지원

Saju Card (공유용):
  - 비율: 1:1 또는 9:16 (인스타 스토리)
  - 그라데이션 배경 (오행 색상 기반)
  - 중앙: 일간 한자 대형
  - 하단: 성격 키워드, 오행 차트
  - 워터마크: 앱 로고
```

## 8. SNS 공유 카드 디자인

### 카드 유형

#### 1) 내 사주 카드
```
크기: 1080x1080 (인스타 피드) / 1080x1920 (스토리)
구성:
  - 상단: 앱 로고 + "나의 사주 분석"
  - 중앙: 일간 한자 대형 + 성격 유형명
  - 중하단: 오행 비율 미니 차트
  - 하단: 성격 키워드 3개 + QR코드(앱 다운로드)
배경: 오행 메인 색상 그라데이션
```

#### 2) 궁합 카드
```
크기: 1080x1080
구성:
  - 상단: "우리의 궁합"
  - 좌측: 내 오행 미니 차트
  - 중앙: 궁합 점수 (큰 숫자) + 하트
  - 우측: 상대 오행 미니 차트 (블러 또는 ? 표시)
  - 하단: "궁합 확인하러 가기" + QR코드
바이럴 포인트: 상대 정보는 가려져 있어서 "누구랑 궁합 봤어?" 궁금증 유발
```

#### 3) 오늘의 운세 카드
```
크기: 1080x1350
구성:
  - 상단: 날짜 + "오늘의 운세"
  - 중앙: 운세 요약 텍스트 (3줄)
  - 하단: 행운 아이템/색상/숫자
  - 배경: 해당 일의 오행 색상
데일리 변화: 매일 다른 디자인 → 매일 공유 동기
```

### 공유 최적화
- 이미지 생성: Flutter의 `RepaintBoundary` + `toImage()` 활용
- 공유 채널별 최적화:
  - 카카오톡: 링크 미리보기 (Open Graph 태그)
  - 인스타그램 스토리: 스티커 포맷 (배경 투명 PNG)
  - 인스타그램 피드: 1:1 정사각형
  - TikTok: 9:16 세로
- 앱 다운로드 딥링크 삽입: 공유 카드 → 앱 설치 → 해당 기능으로 바로 이동

## 9. 애니메이션 & 마이크로인터랙션

### 원칙
- **Purpose > Decoration**: 모든 애니메이션은 목적이 있어야 함
- **Duration**: 200-400ms (일반), 600-1000ms (특별 이벤트)
- **Easing**: `Curves.easeOutCubic` 기본, 바운스는 특별한 경우만
- **Reduce Motion**: 시스템 설정 존중

### 핵심 애니메이션

#### 사주 분석 로딩
```
1단계 (0-1s): 오행 파티클이 화면 중앙으로 모임
2단계 (1-2s): 파티클이 원형으로 정렬 → 오행 차트 형성
3단계 (2-3s): 차트에서 결과 카드로 전환 (fade + scale)
```

#### 매칭 성사
```
1단계: 두 프로필 사진이 좌우에서 중앙으로 이동
2단계: 만나는 순간 하트 파티클 폭발
3단계: "매칭 성공!" 텍스트 + 궁합 점수 페이드인
4단계: "메시지 보내기" CTA 바운스
```

#### 카드 스와이프
```
우측 스와이프 (좋아요):
  - 카드가 우측으로 회전 이동 (rotation + translation)
  - 하트 아이콘 페이드인 (카드 위)
  - 초록 오버레이 점진적 표시

좌측 스와이프 (패스):
  - 카드가 좌측으로 회전 이동
  - X 아이콘 페이드인
  - 빨간 오버레이

위 스와이프 (슈퍼좋아요):
  - 카드가 위로 올라감
  - 별 파티클 이펙트
  - 보라색 오버레이
```

#### 오행 전환
```
프로필 탐색 시 각 사용자의 오행에 따라:
  - 배경 그라데이션 부드럽게 전환
  - 오행 아이콘 회전 등장
  - duration: 300ms, easing: easeInOut
```

### 마이크로인터랙션
```
좋아요 버튼 탭:
  - scale: 1.0 → 1.2 → 1.0 (200ms)
  - 하트가 빨간색으로 채워지는 애니메이션

메시지 전송:
  - 메시지 버블이 하단에서 위로 슬라이드 (150ms)
  - 전송 완료 시 체크 아이콘 (100ms)

Pull to refresh:
  - 오행 심볼이 회전하며 로딩
  - 완료 시 부드럽게 사라짐

탭 전환:
  - 아이콘 scale + color 변화 (200ms)
  - 인디케이터 슬라이드 (300ms)
```

## 10. 화면 구성 (정보 아키텍처)

### 탭 구조
```
[홈/추천]  [탐색]  [채팅]  [사주]  [프로필]
```

### 홈/추천 탭
- 오늘의 추천 매칭 카드 (스택)
- 오늘의 운세 배너 (상단)
- 좋아요/패스 스와이프

### 탐색 탭
- 근처 사용자 그리드
- 필터 (나이, 거리, 오행 등)
- 최근 활동한 사용자 우선

### 채팅 탭
- 매칭 목록 (대화 시작 전)
- 채팅 리스트 (시간순)
- 읽지 않은 메시지 뱃지

### 사주 탭
- 내 사주 분석 (상세)
- 오늘의 운세
- 대운/세운 타임라인
- 사주 카드 공유 버튼

### 프로필 탭
- 내 프로필 보기/편집
- 설정
- 구독 관리
- 공지사항/고객센터
